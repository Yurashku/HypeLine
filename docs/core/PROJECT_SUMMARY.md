# Project Summary

## Главная цель

HypeLine — система поддержки продуктовых улучшений и пилотов. Она помогает пройти путь от сырой идеи до накопленного знания, которое можно использовать в следующих гипотезах.

Система не должна заменять людей в принятии продуктовых решений. Ее роль — структурировать работу: уточнять гипотезы, помогать выбирать метрики, собирать основания, ранжировать идеи, поддерживать разбор пилотов и сохранять знания.

## Текущий контур

```text
сырая идея
  -> уточняющий диалог с пользователем
  -> проверяемая гипотеза
  -> набор метрик
  -> основания и аналоги
  -> оценка качества и приоритета гипотезы
  -> пилот
  -> разбор результата
  -> запись в память пилотов
  -> поддержка следующих гипотез
```

## Что уже сделано

1. Сформирована рабочая логика проекта: от идеи до памяти пилотов.
2. Отказались от лишнего уровня `Capability Block` в пользу задач и рабочих артефактов.
3. Выделили ИИ-интервьюера как отдельный повторяемый механизм: диалоговое заполнение рабочих форм.
4. Расширили задачу ранжирования гипотез: теперь это не только ожидаемый средний эффект, но многокритериальная оценка качества и приоритета гипотезы.
5. Подготовили короткий портфель задач для встречи с исследовательской командой.
6. Сформировали структуру репозитория как долговременной памяти проекта.
